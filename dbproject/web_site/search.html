Advanced Search


Please select the one or more of the tables shown in the list below to activate it. Then you can choose which fields from each table you would like to include in the final results and which filters should be used to get the desired output. By clicking on the Display checkbox at the left side of each field you can select it or deselect it for displaying in the final output. Moreover, by using the text field Filter and the checkbox NOT, you can set up appropriate filters to get the output of your choice.


The Advanced Search tool provides you will a lot of flexibility, since it enables you to combine different filters and fields in order to get the desired results. The server translates your choices into an SQL query which is subsequently executed on the server. The SQL query itself and the output of its execution will be displayed in a new page. If you are not completely satisfied with the query, you can modify it and use it in the SQL Query tool below.

SQL Query


Using the text box below you can enter your own SQL query, which will be subsequently executed in our MySQL server and the results will be presented in a separate html page. You can save this page either as a text file or as an html file, so that you can examine the results of the query in your own time. If there is an error in your SQL syntax, you will receive the exact MySQL error message so that you can correct your query. This text field works in the same way as an interactive MySQL prompt, but please use it with caution.

Its main functionality is to list the contents of each table in the KiPhoDB database. Construct Phylogenetic Trees


This tools can be used to construct phylogenetic trees based on the protein information that is stored inside KiPhoDB. These trees are very useful when comparing proteins that have similar structure and function and they can offer valuable insight into the evolutionary distance between them. The program ClustalW is used in the background in order to perform the multiple sequence alignment between the amino acid sequences that the user provides as input and the resulting phylogenetic tree is displayed in a new html page.


In order to use this tool, please enter the accession numbers of all proteins of interest, in a comma separated format (e.g. "O08605,O15111,O43318,O43707") and then press the button "Generate".

Browse Cancer/Immune Signalling Pathways


This section allows you to browse manually curated Cancer and Immune signaling pathways in Humans.All the kinase substrate and phosphatase substrate pairs occuring in these pathways can be viewed. There is also an option for viewing the common substrates (which are acted upon by both kinase and phosphatase) in each of these pathways.


The Cancer Signalling Pathways includes various pathways which have been found to cause cancer in humans.It includes many pathways which have been studied extensively by various scientists for example the EFGR1,Wnt,Hedgehog etc.


The Immune Signalling Pathways includes many widely studied immunological pathways for example the Interleukins (IL),TCR(T Cell Receptor),BCR(B Cell receptor) etc.

More info


In order to perform efficient SQL queries, it is important to have a basic understanding of the database schema that KiPhoDB uses. In this section we are going to provide you with some basic information on the ways you can use SQL in order to display the database tables and their fields.


In order to display the tables of KiPhoDB, you can use the SQL query input field above and type in the following command: "SHOW TABLES;". This command will provide you with a list of all tables in the KiPhoDB database. The MySQL user that connects to the database server in the background and perfoms these queries has only the basic permissions of listing the contents of each table. Moreover, the user cannot see some tables that are used for user administration and storage of sensitive data.


If you are interested in a specific table and you would like to view more information about it, you can use the command: "SHOW FIELDS FROM TABLE", where TABLE is the name of the table you have chosen. This command will list all fields of the table, their sizes and their properties.
